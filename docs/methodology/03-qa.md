# Step 3 - QA Checklist

Every item on this list must pass before the site is handed over. No exceptions. "Mostly working" is not done. If an item fails, fix it and re-check that item.

---

## How to Use This Checklist

Work top to bottom. Mark each item PASS or FAIL. Any FAIL stops the handover - fix and re-verify before moving on. A site is ready for handover only when every item reads PASS.

---

## Links and Navigation

- [ ] All navigation links work (no 404s)
- [ ] All in-page anchor links scroll to the correct section
- [ ] Footer links work (privacy policy if present, social links if present)
- [ ] No links still pointing to old platform (Wix, Squarespace, GoDaddy, etc.)
- [ ] No links to placeholder or test pages
- [ ] Logo links back to homepage (if it's a link)

**How to check:** Open DevTools > Console, then click every link on the page. Watch for 404 errors in the Network tab. Also run: `curl -sI [URL]` on any URL you're unsure about.

---

## Phone Number

- [ ] Phone number is visible on desktop
- [ ] Phone number is visible on mobile (above the fold or in sticky nav)
- [ ] Phone number is a `<a href="tel:+1XXXXXXXXXX">` link
- [ ] Tapping the phone number on mobile opens the dialer

**How to check:** Open Chrome DevTools > Toggle Device Toolbar (Ctrl+Shift+M). Load the page at 375px width. Tap the phone number - iOS/Android simulation should show a dialer prompt.

---

## Scheduling Modal / CTA

The scheduling button must appear in all of these locations and must work from each one:

- [ ] Header / sticky navigation pill
- [ ] Hero section (primary CTA button)
- [ ] Services section (button on each service card)
- [ ] Contact section (primary CTA button)
- [ ] Mobile floating action bar (if present - mobile-only pinned bar at bottom)

**For each location:** Click the button. Confirm the modal opens. Confirm the modal can be dismissed (X button, click outside, or Escape key).

## Map - Mobile Shop Sites Only

- [ ] Map renders on the live site (tiles load, not a gray box)
- [ ] Markers appear on the map for each service area city
- [ ] User can scroll over the map without the page locking or jumping

**The scroll test:** On desktop Chrome, hover over the map area and scroll. The page should continue scrolling normally - the map should not "capture" the scroll. If the page freezes while scrolling over the map, the `overflow: clip` fix is missing.

- [ ] Dropdown suggestions in the admin service area input appear ABOVE the map (not behind it)

**The z-index test:** In the admin panel, type a city name in the service area search input. The autocomplete dropdown should appear on top of the map, not disappear behind the map tiles.

## Security Audit - Must Score 100/100

Run the full audit from `workflow/01-audit.md`.

```bash
curl -sI https://DOMAIN.com | grep -i "strict-transport\|content-security\|x-frame\|x-content\|referrer\|permissions\|cross-origin"
```

- [ ] Security score: **100 / 100**

All headers must be present in the response:
- [ ] `Strict-Transport-Security` with `max-age=63072000; includeSubDomains; preload`
- [ ] `Content-Security-Policy`
- [ ] `X-Frame-Options: DENY`
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `Referrer-Policy: strict-origin-when-cross-origin`
- [ ] `Permissions-Policy`
- [ ] `Cross-Origin-Opener-Policy: same-origin`
- [ ] `Cross-Origin-Resource-Policy: same-origin`
- [ ] HTTPS enforced (HTTP redirects to HTTPS)

## Cloudflare SSL Configuration

Run these before anything else - a broken SSL config blocks every visitor before they even see the page.

```bash
curl -so /dev/null -w "%{http_code}" https://DOMAIN.com
curl -so /dev/null -w "%{http_code}" https://www.DOMAIN.com
```

- [ ] `https://DOMAIN.com` returns **200**
- [ ] `https://www.DOMAIN.com` returns **301** (redirect to non-www, NOT 525 or 200)
- [ ] Cloudflare SSL/TLS mode is **Full (strict)** - confirmed in dashboard: domain → SSL/TLS → Overview
- [ ] **Always Use HTTPS** is **On** - SSL/TLS → Edge Certificates
- [ ] **Automatic HTTPS Rewrites** is **On** - SSL/TLS → Edge Certificates
- [ ] `www.DOMAIN.com` is added as a custom domain in Cloudflare Pages - Workers & Pages → project → Custom domains

> **525 on www = custom domain not added to Pages.** Fix: Workers & Pages → project → Custom domains → add `www.DOMAIN.com`. Workers & Pages is account-level - click the Cloudflare logo to get to account home first.

## Final Sign-Off

Before marking the build complete and initiating handover:

1. Run the curl header check - confirm all security headers present
2. Open the live URL in an incognito/private window - confirm the site loads correctly for a fresh visitor with no cookies
3. Re-run the SEO audit - confirm 100/100
4. Confirm every checkbox above is PASS

If all pass: proceed to `workflow/04-handover.md`.
